# CAM-WORKBENCH-AUTHORING-VISION

## Objective

Define the canonical product direction for Workbench Authoring as a future standalone code-first authoring system.

## In scope

- vision doc
- product goals and non-goals
- standalone-product extraction boundaries
- canonical user journey: `Parse -> Enrich -> Inspect -> Tune -> Consolidate -> Persist`
- package-seam decisions for contracts, source analysis, and inspector shell
- accepted execution and ownership rules for the standalone product

## Out of scope

- source graph implementation
- authoring model implementation
- live preview mechanics
- source sync mechanics
