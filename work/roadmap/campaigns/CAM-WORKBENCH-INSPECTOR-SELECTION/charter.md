# CAM-WORKBENCH-INSPECTOR-SELECTION

## Objective

Add the first semantic hover/select overlay flow for Workbench Inspector without introducing editing ownership or app-specific logic into the package.

## In scope

- generic hover/select overlay behavior
- minimal sidebar details for the selected target
- first semantic app target markers
- app-side target registry and workbench bridge stubs

## Out of scope

- editing runtime values
- full ownership explorer
- production exposure
