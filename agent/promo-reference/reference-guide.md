# Promo Reference Guide

Этот pack нужен для работы над `/promo`.

Его задача:
- не держать реф "в голове";
- сверяться с ним по фиксированным сценам;
- после каждой итерации проверять не идеальность, а направление.

## Что берем из каждой сцены

### `01-hero`

Берем:
- один главный фокус на первом экране;
- большой headline;
- screenshot или product-visual как главный объект, а не как вложение в карточку;
- очень мало самостоятельных поверхностей.

Не копируем буквально:
- брендовые цвета;
- точную шапку;
- exact iconography и product diagram.

### `02-story-a`

Берем:
- крупный ритм секций;
- чередование `text-left / visual-right`;
- много воздуха вокруг смыслового блока;
- ощущение одной сцены, а не списка карточек.

Не копируем буквально:
- чужие charts;
- чужую taxonomy UI;
- пропорции иллюстраций один-в-один.

### `03-story-b`

Берем:
- большие product scenes вместо россыпи мелких box-элементов;
- спокойный переход между narrative beats;
- ограниченное число акцентов внутри одного экрана.

Не копируем буквально:
- набор feature-cards;
- конкретный visual set иконок;
- spacing пиксель-в-пиксель.

### `04-proof-cta`

Берем:
- редкие акцентные секции;
- отдельные proof-блоки без визуального шума;
- подготовку к финальному CTA через более спокойную композицию.

Не копируем буквально:
- сторонние integration logos;
- их security/enterprise imagery;
- конкретные карточки с proof-content.

### `05-footer`

Берем:
- один сильный CTA перед футером;
- спокойное завершение страницы;
- простой, не шумный footer.

Не копируем буквально:
- текст CTA;
- структуру footer columns;
- exact footer branding.

## Что считаем уходом не в ту сторону

Плохие сигналы:
- секция распадается на много равноправных карточек;
- screenshot выглядит как маленькая картинка внутри панели;
- на одном экране спорят headline, video, screenshot и CTA;
- акцентный цвет используется почти на каждом блоке;
- лендинг выглядит как фрагменты dashboard, а не как маркетинговая лента.

## Review loop для каждой следующей итерации

После каждой волны изменений по `/promo`:
1. снять свежий full-page screenshot страницы;
2. сравнить его с `master.png` и relevant slices;
3. пройти [comparison-checklist.md](./comparison-checklist.md);
4. коротко записать:
   - что приблизилось к рефу;
   - что все еще остается слишком карточечным;
   - какой slice будет основным ориентиром для следующей волны.

Рабочее правило:
- не делать абстрактное "улучшим promo";
- каждая новая итерация должна быть направлена на один из векторов:
  - hero alignment;
  - story rhythm alignment;
  - showcase scale alignment;
  - cta/footer alignment.
