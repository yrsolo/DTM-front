# Docs Archive

`docs/archive/` хранит технические документы, которые больше не являются текущим source of truth.

Сюда попадают материалы, которые:
- описывают снятые или заменённые модули;
- фиксируют старый routing/runtime contract;
- нужны только как historical/reference context.

Правила:
- актуальная документация живёт только вне `docs/archive/`;
- архив не должен использоваться как инструкция для текущих изменений;
- process history и campaign artifacts продолжают жить в `work/archive/`, а не здесь.

Если документ важен для текущей эксплуатации или разработки, он не должен переезжать в `docs/archive/`.

На текущем этапе архивный слой может быть почти пустым. Это нормально: лучше держать `docs/` чистым, чем переносить туда-обратно current docs ради формальности.
