# Workbench Inspector Overview

`Workbench Inspector` — это локальный dev-only инструмент для навигации по React-структуре живого интерфейса и быстрой привязки видимых зон к текущему workbench.

Он нужен не для редактирования страницы “как в конструкторе”, а для того, чтобы:

- быстро выбрать нужный элемент на странице;
- увидеть его место в иерархии;
- понять, есть ли у него semantic enrichment;
- отметить важные элементы и скрыть остальной шум;
- перейти в текущий workbench-контур, если для узла есть bridge.

## Где доступен

Inspector работает только локально и только при явном opt-in.

- нужен `dev`-режим приложения;
- включение идёт через query flag `?inspector=1`;
- флаг сохраняется локально, поэтому inspector может включаться и в следующих reload-сессиях, пока его не выключить через `?inspector=0`.

## Из чего состоит UI

- `launcher` — узкая плавающая кнопка, которая всегда доступна при включённом inspector;
- `shell` — развёрнутая панель inspector;
- левый блок `Layers` — дерево React-узлов страницы;
- правый блок `Properties` — свойства выбранного узла;
- режимы `All` и `Focus`;
- переключатель `Pick mode`;
- фильтр `Smart DOM / Raw DOM`.

## Как пользоваться

Обычный сценарий:

1. Открыть страницу с `?inspector=1`.
2. Нажать на launcher и развернуть inspector.
3. Выбрать узел в дереве или включить `Pick mode`.
4. Посмотреть подсветку элемента на странице.
5. Проверить свойства справа.
6. При необходимости отметить важные элементы.
7. Переключиться в `Focus`, чтобы скрыть остальной шум и оставить только нужные ветки.

## Что показывают режимы

### `All`

Показывает всё текущее дерево зарегистрированных React-узлов.

### `Focus`

Показывает только отмеченные важные узлы и их родительский путь, чтобы не терять контекст.

### `Pick mode`

Переводит inspector в режим выбора мышкой на самой странице.

- интерфейс страницы временно неактивен;
- сама панель inspector остаётся активной;
- наведение показывает рамку на странице;
- клик выбирает узел и синхронизирует его с деревом.

### `Meaningful components`

Показывает более полезную и сжатую картину дерева React-узлов.

### `All registered nodes`

Показывает более полный runtime graph зарегистрированных React-узлов.

## Что inspector показывает справа

Для выбранного узла inspector может показать:

- базовые свойства узла;
- размеры и позицию;
- тип узла и DOM tag;
- semantic enrichment, если он есть;
- ownership hints;
- bridge-действия вроде перехода в workbench.

Если semantic mapping нет, inspector всё равно работает как универсальный React-first инструмент.

## Что inspector не делает

- это не production feature;
- это не второй editor;
- это не page builder;
- это не источник правды для design values;
- это не полноценная замена текущему workbench.

Source of truth для editable design/runtime values остаётся у текущего workbench.
