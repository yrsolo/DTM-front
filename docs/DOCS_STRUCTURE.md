# Documentation Structure

Canonical documentation paths:
- Product: `docs/product/PRODUCT.md`
- System docs: `docs/system/*`
  - `ARCHITECTURE.md`
  - `API_CONTRACT.md`
  - `MVP_PLAN.md`
  - `ROADMAP.md`
  - `UI_DESIGN_CONTROLS.md`
  - `GEMINI.md`
  - `MASTER_SPEC.md` (historical reference)

Top-level files in `docs/*.md` for these topics are compatibility pointers only.

Rules:
1. New edits should be done only in canonical files.
2. Keep pointer files short and stable.
3. Prefer linking canonical paths in all new documents.
